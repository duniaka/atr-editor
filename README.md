# ATR Editor

An interactive editor for **ISO/IEC 7816-3 Answer-to-Reset (ATR)** byte sequences. Paste or type a hex string and the field editors update automatically — or edit individual fields and watch the hex string rebuild with live `TCK` checksum recalculation.

> Generated by [V0](https://v0.dev) and [Claude AI](https://claude.ai).

## Features

- **Two-way editing** — type a raw ATR hex string and the structured field editors update, or edit individual fields and the hex string rebuilds live.
- **Live `TCK` checksum** — the XOR check byte is recalculated automatically as bytes change.
- **Full interface-byte decoding** — dedicated editors for `TS`, `T0`, `TA1`, `TA2`, `TC1`, `TC2`, `TDi`, and the protocol-specific `TA3`/`TB3`/`TC3` bytes, plus historical bytes.
- **Protocol-aware fields** — group-3 bytes are interpreted from the preceding `TDi` protocol type. For `T=1`, `TA3` decodes as `IFSC`, `TB3` as `BWI`/`CWI`, and `TC3` as LRC/CRC redundancy-code selection.
- **Inline validation** — flags reserved and inconsistent values against ISO/IEC 7816-3:2006 rules.
- **Quick examples** — one-click sample ATRs (minimal T=0, direct convention, T=1 with TCK, EMV-like, and a T=1 IFSC/BWI card).

## Getting started

Install dependencies and start the dev server:

```bash
pnpm install
pnpm dev
```

Then open [http://localhost:3000](http://localhost:3000).

| `pnpm lint` | Run ESLint |

## Project structure

```
app/                        Next.js App Router entry (layout, page, globals)
components/
  atr/                      ATR editor UI
    AtrEditor.tsx           Top-level editor composing the field sections
    AtrHexInput.tsx         Raw hex string input
    AtrHexDisplay.tsx       Rendered/rebuilt hex output
    AtrValidation.tsx       Inline validation messages
    editors/                Per-field editors (TS, TA1, TDi, TA3, TB3, TC3, ...)
  ui/                       shadcn/ui primitives
lib/
  atr.ts                    ATR parsing, encoding, and checksum logic
  utils.ts                  Shared helpers
```

## Background

An ATR is the first response a smart card sends after a reset, describing the
communication parameters it supports. The bytes are position- and
protocol-dependent: the meaning of interface bytes such as `TA3` and `TB3`
depends on the protocol type `T` encoded in the preceding `TDi` byte.
